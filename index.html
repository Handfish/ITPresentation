1. Apps: Delivering on Core Responsibilities

What is it?

  • Consistently delivering on all defined job responsibilities for application development and maintenance.
  • Providing critical support by maintaining legacy apps, fixing complex issues, and contributing handwritten solutions.
  • Leading development on key applications

What is the impact?

  • Ensured operational stability and continuity of critical department services.
  • Served as the consistent, reliable development resource through periods of team turnover.
  • Delivered new features and applications as required by the department's mission.

Who has it helped?

  • The Department: Has been able to rely on its applications to function correctly and consistently.
  • End Users: Have had uninterrupted access to services and support for their issues.
  • The Team: Benefited from a stable foundation of maintained legacy systems.
This is the foundation. On top of delivering in my defined role, I saw opportunities to improve the very way we work. The next four topics are the strategic improvements I initiated.

CMSC Marketplace

Workshop & Registration Management

  • Public & Private Listings: Searchable workshops for both general public and specific districts.
  • Detailed Info: Includes description, dates, times, location, capacity, and registration status.
  • Shopping Cart & Registration: Add multiple workshops to a cart and register in one form; handles full-capacity scenarios.
  • Roster & Capacity Tracking: Real-time tracking, with downloadable roster PDFs for authorized users.

District Contract & Quote Management

  • Contract Creation: Manage district contracts and private workshop terms.
  • Quote Generation: Bundle multiple workshops and specify costs.
  • Quote-to-Invoice: Convert accepted quotes to invoices seamlessly.
  • Communication: Email contracts and quotes directly to district contacts.

Financial, Admin & Role Management

  • Invoicing & Payments: Automated invoices, payment tracking (POs, checks), and payment application interface.
  • Facilitator & Expense Management: Assign facilitators, track contracts/payments, and manage external workshop expenses.
  • Downloadable Docs: PDFs for invoices, rosters, and records.
  • Role-Based Access: Guest/public users vs. admin staff with a comprehensive dashboard for full marketplace control.

Makerspace Access System

Secure Access & Admin Control

  • RUID card/fob sign-in via a custom kiosk application.
  • Dynamic entry form for tracking visit purposes, fully managed by admins.
  • Role-based permissions for Staff vs. Super Admins.
  • Granular control over admission rules for different university user types.

Live Monitoring & Real-time Data

  • A "Live Access System" dashboard shows entry events as they happen.
  • Utilizes WebSockets to push real-time data to all connected staff clients for instant updates.
  • Features a staff chat log for seamless team communication.
  • Logs every access attempt (granted and denied) for a complete audit trail.

University Reporting & Impact

  • Generates detailed historical reports from the Oracle student database.
  • Enriches sign-in data with student information to filter by major, school, and gender.
  • Provides critical analytics for demonstrating the Makerspace's usage and impact.
  • Delivers at-a-glance daily access statistics for immediate insights.

Custom Data Tables & Filtering

Interactive Visualizations

Platform Features & User Tools

  • Export and download generated tables, charts, and raw map data for offline use.
  • Provides a single, centralized source for comprehensive historical and current NJ statistics.
  • Administrative backend for managing datasets, indicators, and variables.

Embark: HR Onboarding & Survey Platform

Candidate & Job Management

  • Centralized Dashboard: At-a-glance view of all active candidates, key metrics, and onboarding pipeline status.
  • Job Listing Control: Create, manage, and filter job listings with specific requirements for each role.
  • Candidate Auditing: A complete workflow for tracking candidates from initial entry through reference checks to final onboarding.
  • Role-Based Access: Granular permissions for HR staff and Supervisors ensure secure and appropriate data access.

Automated Reference Surveys

Dynamic Offer Letter Generation

CECC Web Scraper & Content Validator

What is it?

  • An automated system that mass-scrapes the Continuing Education (CECC) website.
  • Intelligently parses HTML to extract structured data like course subjects, program titles, and news articles.
  • Compiles all information into a comprehensive master list in a database.

What is the impact?

  • Automated Content Auditing: Replaces hundreds of hours of manual page checking, eliminating human error.
  • Difference Detection: Automatically compares new scrapes against the master list to instantly flag any changes, additions, or deletions.

Who has it helped?

  • Content Managers: Receive automated reports on content changes instead of manually searching for them.
  • The Department: Maintains a higher standard of accuracy and reliability for its public-facing web presence.
  • Students & Public: Experience a more trustworthy and up-to-date website with fewer errors or outdated listings.

Course Listings App

What is it?

  • A centralized application for displaying course information on the web.
  • Features a robust data pipeline process runner built around Meltano ELT.
  • Automatically extracts course data from two distinct systems: the credit course database (Oracle) and the non-credit system (MS SQL Server).
  • Provides an admin interface with bespoke controls to dynamically filter and configure how courses are displayed in tables.

What is the impact?

  • Eliminates Manual Data Entry: The automated ELT process replaces tedious, error-prone manual updates, saving countless hours of staff time.
  • Ensures Data Accuracy: Course listings on the website are always up-to-date, reflecting the latest information from the source databases without delay.
  • Empowers Admins: The flexible filtering options allow non-technical staff to create custom course tables for specific audiences without developer help.

Who has it helped?

  • Content & Program Managers: Are empowered to create and update their own course listings instantly, without waiting for IT intervention.
  • The Department: Can present a unified, professional, and consistently accurate catalog of all its educational offerings to the public.
  • Prospective Students: See a clean, comprehensive, and up-to-date course catalog, improving their user experience and trust in the information.

In Defense of Identifying High Performers

The Imperative of High Performers

The principle of elevating top talent is a constant in the calculus of success. From the pioneers of Silicon Valley to the leaders of global industry, a clear consensus emerges: the engine of any great organization is its high performers, and the surest path to growth is their advancement. This philosophy has been the cornerstone of our recent improvements...

Andy Grove (Intel): The legendary CEO of Intel championed "constructive confrontation" and radical candor as tools to identify the strongest ideas and performers. His entire management philosophy was based on a meritocracy where the best performers were given the toughest problems and the greatest responsibilities.

Ben Horowitz (Andreessen Horowitz): In "The Hard Thing About Hard Things," Horowitz writes extensively about the need to properly compensate and promote your best people, even when it's difficult, because they represent the standard of performance and culture you want to build.

Beth Miller (SmallBizClub): Advocates for systematic identification and development of high-potential employees through structured mentoring and stretch assignments.

Bill Gates (Microsoft): Built an empire on a simple premise: "We win because we hire talented people." This implies a culture that continuously nurtures and elevates that talent.

Brian Chesky (Airbnb): Argues that leaders should identify their 'favorites' based on merit and performance, elevating them as the standard of quality for the entire organization.

Chaka Booker & Ruth Gotian (Forbes Contributors): Emphasize that leaders must actively create the conditions for growth through stretch assignments, exposure, and autonomy for their high achievers.

Darren Ryan (DataDrivenInvestor): Argues that organizations that fail to identify and promote their best talent create competitive disadvantages and lose market position.

David Burkus (Author): Promotes transparent performance metrics and clear advancement criteria to ensure high performers are properly recognized and elevated.

Elon Musk (Tesla, SpaceX): Looks for "clear evidence of exceptional ability" in every hire. "It is a mistake to hire huge numbers of people to get a complicated job done. 2. Single Sign On: Modernizing Authentication

What is it?

  • A custom-built, production-ready SSO system that interfaces with Rutgers' central CAS identity provider.
  • Offers rock-solid compatibility with any mainstream programming language.
  • Integrates NetID logins with commercial software like Salesforce.

What is the impact?

  • Unblocked All Modernization: The successful transition to TypeScript was only possible because of this project.
  • Eliminated Language Lock-in: We are no longer constrained to specific technologies for authentication.
  • Centralized & Secure: A single, reliable way to handle NetID logins for all applications.

Who has it helped?

  • The Development Team: Can build in any language without custom CAS integration headaches.
  • The Department: Gained immense flexibility in technology choices and hiring.
  • The Entire University: This is a potential university-wide asset that simplifies development for any department.
This project was born purely from ambition. I saw a university-wide problem and built a solution that provides value far beyond our department.

4. Containerization & CI: Reproducible Builds

What is it?

  • Implementation of Podman container system for Dev, QA, and Production environments.
  • Reproducible builds of legacy applications by containerizing their old dependencies (e.g., specific Java versions).
  • Continuous Integration (CI) pipelines that automatically deploy the latest code to servers.

What is the impact?

  • Eliminated Version Conflicts: New applications no longer break old ones. Each app is perfectly isolated.
  • Solved Legacy Builds: We can now build and run old applications reliably, forever.
  • Enhanced Security & Stability: Vulnerabilities can be isolated to a single container, not the entire server.
  • Automated Workflows: Instant, labor-free deployments for testing and review.

Who has it helped?

  • The Infrastructure Team: Massively reduced workload, fewer dependency issues, and fewer server-level tickets.
  • Developers: Can build and deploy reliably without "it works on my machine" problems.
  • Management & QA: Guaranteed access to up-to-date applications for testing.
Containers solved some of our biggest operational pain points. We no longer live in fear of server-wide updates breaking critical applications.
